## 2. BLOB BACKGROUND ON HERO SECTION

In the hero section only, add 3–4 absolutely positioned blob divs behind the content:
```css
.blob {
  position: absolute;
  border-radius: 50%;
  filter: blur(80px);
```

## 3. INTERSECTION OBSERVER ENTRANCE ANIMATIONS

Create a custom hook `useInView.ts` in `client/src/hooks/`:
```ts
import { useEffect, useRef, useState } from "react";

export function useInView(threshold = 0.2) {
  const ref = useRef<HTMLDivElement>(null);
  const [inView, setInView] = useState(false);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => { if (entry.isIntersecting) setInView(true); },
      { threshold }
    );
    if (ref.current) observer.observe(ref.current);
    return () => observer.disconnect();
  }, []);

  return { ref, inView };
}
```

Apply this hook to each section and add a CSS class when inView is true. Define these animation classes in index.css:
```css
.fade-up {
  opacity: 0;
  transform: translateY(40px);
  transition: opacity 0.6s ease, transform 0.6s ease;
  will-change: transform, opacity;
}
.fade-up.visible {
  opacity: 1;
  transform: translateY(0);
}

.fade-in {
  opacity: 0;
  transition: opacity 0.6s ease;
}
.fade-in.visible {
  opacity: 1;
}

.slide-left {
  opacity: 0;
  transform: translateX(-40px);
  transition: opacity 0.6s ease, transform 0.6s ease;
  will-change: transform, opacity;
}
.slide-left.visible {
  opacity: 1;
  transform: translateX(0);
}

.slide-right {
  opacity: 0;
  transform: translateX(40px);
  transition: opacity 0.6s ease, transform 0.6s ease;
  will-change: transform, opacity;
}
.slide-right.visible {
  opacity: 1;
  transform: translateX(0);
}

.pop-in {
  opacity: 0;
  transform: scale(0.95);
  transition: opacity 0.5s ease, transform 0.5s ease;
  will-change: transform, opacity;
}
.pop-in.visible {
  opacity: 1;
  transform: scale(1);
}
```

Apply animations per section:
- Hero: title words get `fade-up` with inline `transition-delay` incrementing by 100ms per word — split the title string by spaces and map each word to a `<span>`
- How It Works: each step card gets `slide-left` with 150ms staggered delay
- Featured Projects: each project card gets `pop-in` with 100ms staggered delay
- Community section: image gets `slide-left`, text block gets `slide-right`
- FAQ items: each accordion item gets `fade-up` with 80ms staggered delay

## 4. VERTICAL DOT NAVIGATION

Add a `DotNav.tsx` component in `components/`. It renders 7 vertical dots on the right side of the screen (fixed position). Each dot corresponds to a landing page section. The active dot is filled Midnight Plum, inactive dots are outlined. Clicking a dot calls `.scrollIntoView({ behavior: 'smooth' })` on the corresponding section. Track the active section using an IntersectionObserver that watches all 7 sections and updates active index when one is >50% in view.

Place `<DotNav />` inside `Landing.tsx` only, not on other pages.
